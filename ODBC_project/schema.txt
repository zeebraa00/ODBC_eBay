Auction system data schema

<<user data>>

- uid : primary key
- name
- email
- password
- level


<<watch list>>

- uid : foreign key
- id : foreign key


<<item>>

- id : primary key
- uid : foreign key
- category
- description
- condition (new, like-new, very-good, good, acceptable)
- most recent bid price
- buy it now price
- status
- date posted
- bid ending date


<<bid history>>

- uid : foreign key
- id : foreign key, primary key
- bid-price : primary key


<<transaction>>

- tid
- id
- date
- sellers : foreign
- buyers : foreign
- sell-price : foreign
